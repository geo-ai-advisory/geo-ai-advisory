# Поиск кандидатов на hh.ru

## Вызов
`/hh` или `/hh [позиция/задача]`

## Поведение

### Без аргументов (`/hh`)
1. Прочитай `Projects/hh/CONTEXT.md` из корня проекта
2. Покажи статус и спроси:

> Проект "HH - Поиск кандидатов". Что делаем?
>
> - Новый поиск - `/hh [позиция]`
> - Продолжить последний - `/hh продолжить`
> - Посмотреть шортлист - покажу ссылку и статус

### С позицией (`/hh [позиция]`)
1. Прочитай `Projects/hh/CONTEXT.md` - проверь нет ли уже поиска по этой позиции
2. Прочитай `~/.claude/skills/hh-resume-search/SKILL.md` - методология раундового поиска
3. Создай журнал в `journals/ГГГГ-ММ-ДД-hh-[позиция]/log.md`
4. Начни поиск по раундовой методологии:
   - R1-R2: широкие запросы по ролям
   - R3: по названиям компаний из найденных резюме
   - R4: "Похожие резюме" - ML-матчинг
   - R5: целевой поиск "компания + роль"
   - R6-R7: внешние источники, конференции, статьи
5. После каждого раунда - тиринг кандидатов (S/A/B/C)
6. По завершении - HTML-шортлист с шифрованием AES-256-GCM

### Продолжение (`/hh продолжить`)
1. Прочитай `Projects/hh/CONTEXT.md` - определи последний поиск
2. Найди последний журнал в `journals/` с префиксом `hh-`
3. Прочитай `log.md` целиком - пойми где остановились
4. Продолжи с того места (следующий раунд / доп. запросы / обновление шортлиста)

## Методология поиска
Полная техника описана в `~/.claude/skills/hh-resume-search/SKILL.md`. Ключевое:
- Раундовый поиск с углублением (R1-R7+)
- Компании из резюме предыдущего раунда = запросы для следующего
- Блок "Похожие резюме" на hh.ru - золотая жила (ML-матчинг)
- Короткие точные запросы 2-3 слова, НЕ длинные
- Form-based поиск через Playwright, НЕ URL-based с фильтрами

## HTML-шортлист
Шаблон оформления описан в `~/.claude/skills/hh-resume-search/SKILL.md` (секция "HTML-шортлист: шаблон оформления").
- Светлая тема, карточки с цветовой полоской по тиру
- Шифрование AES-256-GCM, деплой на GitHub Pages
- Референс: https://engwatch.github.io/hh-search-shortlist/

## После выполнения
- Обновить `Projects/hh/CONTEXT.md` (новые кандидаты, статусы, история поисков)
- Логировать в `journals/` как обычно
- Промоутировать навыки если найдены новые приемы

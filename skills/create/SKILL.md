# Создание нового проекта

## Вызов
`/create [название]` или `/create`

## Поведение

### Без аргументов (`/create`)
Спроси:
```
Создание нового проекта. Что нужно:
1. Название проекта (латиницей, через дефис)
2. Краткое описание - что это за проект
3. Нужна ли слэш-команда? (например /my-project)
```

### С аргументами (`/create название`)
1. Создай папку `Projects/[название]/` в корне проекта
2. Создай `Projects/[название]/CONTEXT.md`:

```markdown
# [Название] - [описание]

## Описание
[Что это за проект, цель]

## Текущий статус
В работе / Планирование

## Команды
- `/[название]` - интерактивный помощник (если создан скилл)

## Ключевые ресурсы
[Ссылки, spreadsheets, репо - заполнится по ходу работы]

## История
| Дата | Что сделано |
|------|------------|
| [сегодня] | Проект создан |
```

3. Если нужна слэш-команда - создай `~/.claude/skills/[название]/SKILL.md`:

```markdown
---
name: [название]
description: Use when user invokes /[название] or asks about [описание]
---

# [Название проекта]

## Вызов
`/[название]` или `/[название] [задача]`

## Поведение

### Без аргументов
1. Прочитай `Projects/[название]/CONTEXT.md`
2. Покажи статус и спроси что делать

### С аргументами
Распознай намерение и выполни.

## После выполнения
- Обновить CONTEXT.md
- Логировать в journals/ как обычно
```

4. Добавь запись в MEMORY.md (секция проектов)

## Существующие проекты (для справки)
Папка `Projects/` в корне B-project. Каждый проект имеет CONTEXT.md.

## Правила
- Название папки - латиницей через дефис (kebab-case)
- CONTEXT.md обязателен
- Скилл опционален - создавать если проект будет использоваться повторно
- Не дублировать существующие проекты - проверить ls Projects/ перед созданием

# Поиск DM + PM на hh.ru

## Вызов
`/hh-dm-pm` или `/hh-dm-pm [действие]`

## Контекст

Ищем **2 человека**: 1 Delivery Manager + 1 Project Manager для INSAP (InsurTech, партнёрское страхование).

**Delivery Manager (DM)** — 60% партнёрский менеджер + 40% проджект. Запускает партнёров, считает фин. модель, знает ДДС и юр. обвязку.

**Project Manager (PM)** — 70% проджект + 30% бизнес. Мультитрекинг, команда разработки, road map, спринты. Не обязан запускать партнёров лично, но должен понимать страховой рынок.

## Поведение

### Без аргументов (`/hh-dm-pm`)
1. Прочитай `Projects/hh/delivery_hh/CONTEXT.md`
2. Покажи сводку:

> **DM + PM поиск** | Кандидатов: X (DM: Y, PM: Z)
>
> - Продолжить поиск — `/hh-dm-pm поиск`
> - Статус по ролям — `/hh-dm-pm статус`
> - Провести собеседование — `/hh-dm-pm интервью [имя]`
> - Оценить кандидата — `/hh-dm-pm оценка [имя]`
> - Обновить шортлист — `/hh-dm-pm шортлист`

### Статус (`/hh-dm-pm статус`)
1. Прочитай `Projects/hh/delivery_hh/CONTEXT.md`
2. Покажи:
   - Топ-5 DM кандидатов (отсортировано по П-скору)
   - Топ-5 PM кандидатов (отсортировано по Д-скору)
   - Кто активно ищет / срочно контактировать
   - Сколько собеседований проведено, вердикты

### Поиск (`/hh-dm-pm поиск`)
1. Прочитай `Projects/hh/delivery_hh/CONTEXT.md` — проверь что ещё не искали
2. Прочитай `~/.claude/skills/hh-resume-search/SKILL.md` — раундовая методология
3. Создай журнал `journals/ГГГГ-ММ-ДД-hh-dm-pm/log.md`
4. Начни/продолжи поиск
5. При тиринге — ставить П/Д скоры + автоклассификацию роли:
   - П ≥ 7, Д ≥ 6 → DM
   - Д ≥ 7, П < 7 → PM
   - П ≥ 7, Д ≥ 7 → DM/PM
   - П 5-6, Д ≥ 8 → PM+
   - П ≥ 8, Д < 6 → DM-risk
   - Оба < 5 → ОТКАЗ

### Интервью (`/hh-dm-pm интервью [имя]`)
1. Прочитай `Projects/hh/delivery_hh/interview-plan-delivery-manager.md`
2. Прочитай `Projects/hh/delivery_hh/interview-plan.html` — рекрутёрский скрипт
3. Подготовь материалы для Маши (рекрутёр):
   - Карточка кандидата (из CONTEXT.md)
   - Что спрашивать (из интервью-плана)
   - На что обратить внимание по конкретному кандидату

### Оценка (`/hh-dm-pm оценка [имя]`)
Принимает транскрипт/запись собеседования, генерирует:
1. Прочитай интервью-план и систему оценки
2. Рассчитай баллы по ОБЕИМ шкалам (DM и PM)
3. Применяй алгоритм классификации:
   - DM ≥ 40 → **DM**
   - DM 30-39 И PM ≥ 35 → **PM**
   - DM < 30 И PM ≥ 35 → **PM**
   - DM < 30 И PM < 25 → **ОТКАЗ**
   - Стабильность < 2/5 → снижает на 1 ступень
4. Генерируй MD-отчёт в `Projects/hh/delivery_hh/assessment-[имя].md`
5. Предложи `/html-push` для публикации

### Шортлист (`/hh-dm-pm шортлист`)
1. Прочитай CONTEXT.md
2. Сгенерируй HTML-шортлист с двумя секциями: DM и PM
3. Предложи `/html-push` для публикации

## Двойная система оценки

### Скоры
- **П (партнёрский)**: фин. модель, юр., ДДС, переговоры, знание рынка страхования → DM
- **Д (delivery)**: мультитрекинг, структура, верификация, road map, запуск → PM

### Пороги интервью (7 критериев)

| Критерий | Вес DM | Вес PM |
|----------|:---:|:---:|
| Партнёрский опыт | x3 | x1 |
| Фин. модель + ДДС | x2 | x1 |
| Операционные кейсы | x2 | x2 |
| Проджект-навыки | x1 | x3 |
| Стартап-менталитет | x1 | x1 |
| Стабильность/лояльность | x1 | x1 |
| Культурный фит | x1 | x1 |

**Автоотказ DM:** Партнёрский < 9/15 ИЛИ Фин.модель < 6/10
**Автоотказ PM:** Проджект < 9/15
